Question: Describe what data is collected:
Answer (paste this):

This extension collects LinkedIn profile information for prospects that users scan, including:

- Name
- Professional headline
- Current company
- Location
- Number of mutual connections
- Profile URL
- Timestamp of when scanned

User-generated data:
- Status labels assigned by user (e.g., "contacted", "replied")
- Notes written by user about prospects
- Tags created and assigned by user
- Message templates created by user
- Campaign tracking data

Usage statistics:
- Number of profiles scanned
- Last scan date
- Count of new prospects

All data is stored locally on the user's device using Chrome's local storage API. No data is transmitted to external servers or shared with third parties.

Question: Describe how you use the data:
Answer (paste this):

Data is used exclusively to:

1. Display scanned LinkedIn profiles in an organized list within the extension interface
2. Enable users to search, filter, and sort their prospect list
3. Allow users to add notes, tags, and status updates to prospects
4. Track message campaigns and outreach efforts

SECTION 3: DATA TRANSFER
================================================================================

Question: Is any data transferred off the user's device?
Answer: NO

Even though data is local-only, if form requires explanation, use:

Data is stored exclusively in Chrome's local storage on the user's device. No data is transmitted to external servers, third-party services, or the extension developers. The only way data leaves the device is if the user explicitly uses the "Export to CSV" feature to download their data.

================================================================================
SECTION 4: DATA RETENTION AND DELETION
================================================================================

Question: How long is data retained?
Answer (paste this):

Data is retained locally on the user's device indefinitely until the user:
- Deletes individual prospects through the extension interface
- Uses the "Clear All Data" function to delete all data at once
- Uninstalls the extension (which automatically removes all local data)

Users have complete control over data retention and can delete data at any time without contacting support.

---

Question: Can users request data deletion?
Answer: YES, users can delete data themselves through the extension interface at any time

---

Question: How can users delete their data?
Answer (paste this):

Users can delete data in three ways:

1. Delete individual prospects: Click the delete/trash icon next to any prospect in the extension interface

2. Delete all data: Open extension settings and click "Clear All Data" button

3. Complete removal: Uninstall the extension from Chrome, which automatically deletes all stored data

No developer intervention is required. Users have immediate, complete control over data deletion.

SECTION 6: DATA SECURITY
================================================================================

Question: What security measures are in place?
Answer (paste this):

Since all data is stored locally on the user's device using Chrome's secure local storage API:

1. Data never leaves the user's device (except via explicit user export)
2. Chrome's built-in storage encryption protects data at rest
3. No network transmission means no interception risk
4. No external servers means no server-side breach risk
5. Extension code is reviewed by Chrome Web Store security team

Users are advised to:
- Keep Chrome browser updated
- Use strong device passwords
- Secure any exported CSV files appropriately

NOTES FOR DEVELOPER DASHBOARD
================================================================================

When filling out the form:

1. Be consistent: Use the same wording across all fields
2. Be specific: Don't use vague terms like "various data"
3. Emphasize local storage: Repeatedly mention data stays on user's device
4. Clarify no server transmission: Make it clear no backend exists
5. Highlight user control: Users can view, export, and delete data anytime
6. Be honest: If you add features that collect more data, update disclosures

IMPORTANT: If you later add features that collect additional data types or transmit data externally, you MUST update these disclosures and resubmit for review.
